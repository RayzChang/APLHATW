# AlphaTW — 台股 AI 智慧交易助理

> 給新手的 AI 跟單系統 × 給老手的智慧選股工具

**新手請先看 → [完整使用說明書（股市小白版）](docs/USER_GUIDE.md)**

## V2 重構現況

目前專案已完成 V2 主幹重構，系統已經進入「可啟動、可 build、可操作」的狀態。

如果你想用最白話的方式理解：

- `V1`：功能有了，但很多邏輯黏在一起
- `V2`：把分析、決策、交易、風控、排程、API、前端拆成分層結構

### 這次 V2 主要改了什麼？

1. 統一資料模型
   - 分析結果、交易訊號、持倉、投資組合都有固定格式
2. 統一分析流程
   - 單股分析與候選股分析共用同一條 pipeline
3. 拆出候選股篩選與決策引擎
   - 不再把篩選與決策硬塞在 scheduler 裡
4. 拆出交易執行與風控引擎
   - `simulator` 不再一個人包辦下單、風控、帳務
5. 重構排程系統
   - 掃描 job、風控 job、scheduler controller 分離
6. 整理 API 路由
   - 新增較清楚的 `/api/analysis/*` 與 `/api/trading/*`
   - 舊路由仍保留相容層，不會一次全部斷掉
7. 前端開始模組化
   - Dashboard 已拆成 feature components
   - 前端改接新的 V2 API

### V2 帶來的實際進步

- 系統更容易維護，不會一改就牽一大片
- 之後要接真實券商 API、擴充策略、改 UI 都比較容易
- 新舊功能可以共存，不需要一次推翻重做
- 專案現在已能實際啟動並完成前後端驗證

### 目前狀態

- 後端可啟動
- 前端可啟動
- 前端可 build
- 核心分析 / 投組 / 掃描 API 已驗證可用
- 接下來的工作重點是「整理與優化」，不是「救火重建」

## 這個程式在做什麼？

AlphaTW 是一個用 AI 幫你分析台股的輔助工具，核心目標只有一個：

> **幫助新手可以「無腦」跟著 AI 的決定做股票，降低自己研究的門檻。**

它分成三個功能模組，每個人都能找到適合自己的使用方式：

| 功能 | 適合對象 | 核心價值 |
|------|---------|---------|
| AI 模擬交易 | 想跟單的新手 | AI 自動掃描全市場，幫你模擬買賣 |
| AI 回測模擬 | 想先驗證 AI 能力的人 | 用過去 30 天資料驗證 AI 策略的歷史表現 |
| 智慧選股 | 有在看某檔股票的人 | 輸入代號，AI 告訴你要不要買、買在哪 |

### 功能一：AI 模擬交易（自動跟單）

**位置**：首頁 Dashboard → 點擊「執行 AI 交易分析（全市場）」

#### 這個功能在做什麼？

想像你給了 AI **100 萬台幣的虛擬資金**，讓它去股市幫你操盤。  
AI 每天在台股開盤後自動醒來，掃描全市場超過 2,000 支股票，找出最有潛力的標的，然後執行模擬交易。

你不需要懂股票技術分析，只要：
1. 看 AI 今天買了什麼
2. 如果你信任 AI，就跟著它買同樣的股票
3. AI 賣出時，你也賣出

**這就是「跟單」。**

#### 它是怎麼選股的？

AI 使用兩層篩選機制，這樣既快速又準確：

```
第一層：技術指標快速篩選（全部 2,000+ 支股票）
    ↓ 計算 RSI、KD、MACD、均線、布林帶
    ↓ 綜合評分，取前 30 名候選股

第二層：四位 AI 探員深度分析（僅前 30 名）
    ↓ 技術分析師：看 K 線型態、趨勢
    ↓ 情緒分析師：看最新新聞、市場情緒
    ↓ 風控師：評估風險、計算停損停利
    ↓ 首席決策官：綜合以上，做出最終判斷

    只有「信心度 ≥ 70%」且「決定買入」才會下單
```

#### 自動排程時間

AI 不需要你手動觸發，會在以下時間自動執行：

- **每個交易日 09:10**（開盤 10 分鐘後）
- **每個交易日 12:30**（午盤）

持倉期間，AI 每分鐘監控所有持倉，觸發停損或停利時**立即賣出**。

#### 風險控管機制

| 機制 | 觸發條件 | 說明 |
|------|---------|------|
| 停損 | 虧損達 2×ATR | 立即賣出，保護資金 |
| 停利 | 獲利達 3×ATR | 鎖定獲利，賣出 |
| 保本止損 | 獲利超 5% 後，拉回到成本附近 | 避免浮盈變浮虧 |
| 最長持倉 | 持有超過 20 個交易日 | 強制出場，避免長期套牢 |

> **ATR（Average True Range）**：衡量股票日常波動幅度的指標，數字越大代表這檔股票波動越劇烈。

#### LINE 即時推播通知

設定好 LINE Bot 後，AI 的每一步操作都會即時推送到你的 LINE：

- **買入通知**：告訴你買了什麼、多少股、停損停利設在哪
- **賣出通知**：告訴你賣了什麼、原因、賺賠多少
- **掃描完成通知**：告訴你掃了幾支、找到幾個候選、下了幾筆單

> 設定方式見 [快速啟動 — LINE Bot 設定](#22-設定-line-bot選用)

#### 交易紀錄匯出 CSV

在「歷史交易紀錄」區塊右上角點擊「匯出 CSV」，即可下載所有買賣紀錄為 Excel 可開啟的表格檔案，方便記帳或對帳。

#### 在 Dashboard 可以看到什麼？

- **虛擬總資產**：100 萬 + 獲利或虧損
- **AI 決策日誌**：每一筆買賣記錄，包含原因
- **當前持倉庫存**：現在持有哪些股票、獲利狀況
- **AI 系統運作說明**：上次掃描時間、掃了幾支股票、找到幾個候選

### 功能二：AI 回測模擬（歷史驗證）

**位置**：首頁 Dashboard → 下滑找到「AI 回測模擬」區塊 → 展開

#### 這個功能在做什麼？

在你決定信任 AI 跟單之前，你一定想知道：

> **「這個 AI 如果在過去 30 天操盤，結果會怎樣？」**

回測模擬就是回答這個問題的工具。  
它用跟「AI 模擬交易」**相同的買賣邏輯**，在歷史資料上重播一遍，告訴你 AI 的歷史績效如何。

#### 兩種回測模式

**精選標的模式（快速）**
- 掃描台股各產業龍頭約 50 支
- 使用 FinMind 歷史 K 線
- 約 60~120 秒完成

**全市場掃描模式（真實）**
- 掃描全部台股 TWSE + TPEX 約 2,000+ 支普通股
- 與 AI 即時交易系統邏輯完全一致
- 約 3~5 分鐘完成

#### 回測結果怎麼看？

| 指標 | 說明 |
|------|------|
| **總報酬率** | 100 萬變成多少，比例呈現 |
| **總損益** | 實際賺了或虧了多少元 |
| **勝率** | 這段時間的交易，幾成是賺的 |
| **最大回撤** | 過程中最大虧損幅度（越小越好） |
| **盈虧比** | 平均每賺 1 元，需要承擔多少虧損風險 |
| **0050 基準** | 同期直接買 0050 ETF 的報酬，用來對比 AI 的表現 |

#### ⚠️ 重要說明

回測使用「技術指標評分」模擬買賣決策（免費、快速）。  
而真實 AI 即時交易系統在技術篩選後，還會呼叫四個 Gemini AI Agent 進行深度精析，兩者的買賣時機點會有差異。

**回測的用途是：**
- 驗證技術指標策略在這段歷史上是否能獲利
- 了解這個市場環境下 AI 系統大致的績效水平
- **並非精確預測 AI 每一筆交易的結果**

### 功能三：智慧選股（單股深度分析）

**位置**：上方導覽列 → 「智慧選股」分頁

#### 這個功能在做什麼？

假如你最近在關注某檔股票（比如台積電 2330），想知道：

- **現在適不適合買？**
- **買進的話應該買在哪個價位？**
- **停損設在哪裡？萬一跌了我應該在哪裡認賠？**
- **目標價是多少？漲到哪裡可以獲利了結？**

只要輸入股票代號，四位 AI 探員會在 1~2 分鐘內給你一份完整分析報告。

#### 四位 AI 探員分別做什麼？

| 探員 | 分析內容 |
|------|---------|
| 📈 技術分析師 | K 線型態、RSI、MACD、均線、布林帶、支撐壓力位 |
| 🎭 情緒分析師 | 近期相關新聞、市場輿論、社群討論風向 |
| 🛡️ 風險控管師 | 合理停損位、倉位大小建議、風險報酬評估 |
| 👔 首席決策官 | 綜合三份報告，做出「買入 / 觀望 / 賣出」最終決策 |

#### 分析結果包含什麼？

- **AI 最終決策**：BUY（買入）/ HOLD（觀望）/ SELL（賣出）
- **AI 信心百分比**：AI 對這個決策有多確定
- **進場參考價**：當前即時股價
- **止盈目標價**：漲到這裡可以獲利了結
- **止損防護價**：跌到這裡應該賣出認賠
- **建議倉位比例**：建議投入你資金的幾個百分比

#### 重要提醒

> **這個功能的最終決定權在你手上！**
>
> AI 提供的是參考意見，不是強制指令。  
> 建議結合自己對公司的了解、市場大環境，再做最終決定。
>
> 股市有風險，任何分析工具都無法保證 100% 獲利。

## 快速啟動

### 1. 安裝相依套件

```bash
# Python 後端
pip install -r requirements.txt

# 前端
cd frontend
npm install
```

### 2.1 設定環境變數（必要）

複製 `.env.example` 為 `.env`，填入以下必要金鑰：

```
GEMINI_API_KEY=你的_Gemini_API_金鑰
FINMIND_TOKEN=你的_FinMind_API_Token
```

- **Gemini API Key**：[Google AI Studio](https://aistudio.google.com/) 免費申請
- **FinMind Token**：[FinMind 官網](https://finmindtrade.com/) 免費申請（每小時 600 次免費額度）

### 2.2 設定 LINE Bot（選用）

如果希望 AI 買賣時即時推送 LINE 通知：

1. 前往 [LINE Developers](https://developers.line.biz/)，用 LINE 帳號登入
2. 建立一個 **Messaging API** 頻道
3. 在頻道設定頁面找到 **Channel access token**，點「Issue」產生
4. 用你的 LINE 掃描 Bot 的 QR Code，加 Bot 為好友
5. 取得你的 **User ID**（在 LINE Developers 頻道的「Basic settings」頁面底部）
6. 在 `.env` 填入：

```
LINE_CHANNEL_ACCESS_TOKEN=你的頻道存取權杖
LINE_USER_ID=你的LINE用戶ID
```

7. 重啟後端，到系統設定（齒輪圖示）開啟 LINE 推播開關

### 3. 啟動後端

```bash
uvicorn api.app:app --host 0.0.0.0 --port 8000 --reload
```

### 4. 啟動前端（開發模式）

```bash
cd frontend
npm run dev
```

前端預設在 `http://localhost:5173` 開啟。

### 5. 前端正式 build

```bash
cd frontend
npm run build
```

補充：

- 建議使用英文專案路徑，例如 `C:\Users\Admin\Desktop\Code\APLHATW`
- 在 Windows 中文路徑下，部分 Vite / Node 組合可能會出現 build crash
- 目前專案已在英文路徑完成 build 驗證

## 常見問題

**Q：回測結果為什麼交易次數是 0？**

可能原因：
1. FinMind Token 未設定或已超過每小時 600 次額度（精選模式）
2. 選擇「全市場模式」時，AI 掃描排程正在執行中（需等待 3~5 分鐘）
3. 當前市場環境技術指標條件太嚴，沒有任何股票達到買入標準（可嘗試調低「買入門檻」到「≥1 積極」）

---

**Q：AI 分析要花多少錢？**

- **技術指標篩選（第一層）**：完全免費，不消耗 API 費用
- **四 AI Agent 深度分析（第二層）**：每支股票約 0.002 美元，每次全市場掃描約 0.06 美元
- **智慧選股單次分析**：約 0.008 美元

---

**Q：台股什麼時候可以交易？**

台灣股市交易時間：
- **週一至週五**，上午 **09:00 ~ 下午 13:30**
- 國定假日、颱風假 休市

AI 自動掃描排程：**09:10** 和 **12:30**（僅交易日）

---

**Q：我可以使用真實資金嗎？**

目前系統**僅支援模擬交易**，所有金額都是虛擬的，不會動到你的真實資金。  
待模擬績效驗證穩定後，未來版本將支援串接真實券商 API 進行實盤交易。

---

**Q：智慧選股和 AI 模擬交易有什麼不同？**

| 比較項目 | AI 模擬交易 | 智慧選股 |
|---------|------------|---------|
| 股票範圍 | AI 自動掃描全市場 | 你指定特定股票 |
| 決策者 | AI 完全自主決定 | AI 給建議，你自己決定 |
| 適合對象 | 想無腦跟單的新手 | 有在關注特定股票的人 |
| 執行交易 | 自動模擬下單 | 不執行，僅分析 |
